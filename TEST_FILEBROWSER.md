# FileBrowser 功能测试清单

## 测试步骤

### 1. 打开应用
- 启动 NeoView 应用
- 确认侧边栏显示 "Folder" 标签

### 2. 选择文件夹
- [ ] 点击"选择文件夹"按钮
- [ ] 选择一个包含子文件夹和图片的目录
- [ ] 确认顶部面包屑导航显示完整路径
- [ ] 确认文件列表显示目录内容

### 3. 测试文件夹导航
- [ ] 点击一个子文件夹
- [ ] **检查控制台输出**:
  ```
  === openFile called ===
  Item: { name: "子文件夹名", isDir: true, path: "完整路径" }
  📁 Opening directory: 完整路径
  🚀 navigateToDirectory called with path: 完整路径
  📂 loadDirectory called with path: 完整路径
  🔄 Calling FileSystemAPI.browseDirectory...
  ✅ Loaded X items: [...]
  ✅ Directory navigation completed
  ```
- [ ] 确认进入子文件夹
- [ ] 确认面包屑更新显示新路径

### 4. 测试面包屑导航
- [ ] 点击面包屑中的上级目录
- [ ] 确认返回到该目录
- [ ] 点击不同的面包屑段
- [ ] 确认可以快速跳转

### 5. 测试返回按钮
- [ ] 在子目录中点击"返回上一级"按钮(←)
- [ ] 确认返回到父目录
- [ ] 按 Backspace 键
- [ ] 确认也能返回上一级

### 6. 测试切换标签
- [ ] 切换到 "History" 标签
- [ ] 切换回 "Folder" 标签
- [ ] **确认之前浏览的路径和状态保持不变**

### 7. 测试压缩包
- [ ] 点击 ZIP 或 CBZ 文件
- [ ] 确认显示压缩包内容
- [ ] 顶部显示"压缩包模式"
- [ ] 返回上一级退出压缩包模式

### 8. 测试键盘导航
- [ ] 使用方向键上下选择
- [ ] 按 Enter 打开选中的文件夹
- [ ] 按 Backspace 返回
- [ ] 按 F5 刷新

### 9. 测试侧边栏调整
- [ ] 鼠标移到侧边栏右边缘
- [ ] 看到分隔条高亮
- [ ] 拖拽调整宽度
- [ ] 确认宽度保持

## 预期结果

✅ 所有测试项都应该通过
✅ 点击文件夹能进入子目录
✅ 面包屑导航正常工作
✅ 切换标签后状态保持
✅ 控制台没有错误
✅ 用户体验流畅

## 如果失败

1. 打开浏览器开发者工具(F12)
2. 查看 Console 标签
3. 复制完整的错误信息
4. 提供给开发者
