# 缩略图系统测试

这个目录包含了用于测试 NeoView 缩略图系统的工具和组件。

## 文件说明

### `thumbnail-test.ts`
独立的 TypeScript 测试脚本，可以在 Node.js 环境中运行或在浏览器中调用。

### `ThumbnailSystemTest.svelte`
Svelte 组件形式的测试界面，可以在应用中直接使用。

### `thumbnail-test/+page.svelte`
测试页面路由，可以通过 `http://localhost:1420/thumbnail-test` 访问。

## 测试内容

测试脚本会验证以下功能：

1. **初始化缩略图管理器** - 测试系统的初始化
2. **生成文件缩略图** - 测试从图片文件生成缩略图
3. **生成文件夹缩略图** - 测试从文件夹生成缩略图（递归查找子目录中的图片）
4. **获取缩略图信息** - 测试获取缩略图的详细信息（尺寸、URL等）
5. **缩略图缓存** - 测试缓存机制是否正常工作
6. **清理缩略图** - 测试清理过期缩略图的功能

## 使用方法

### 在应用中测试

1. 启动 NeoView 应用
2. 在浏览器中访问 `http://localhost:1420/thumbnail-test`
3. 点击"运行所有测试"按钮
4. 查看测试结果

### 独立脚本测试

```typescript
import { runThumbnailTests } from './thumbnail-test';

// 运行所有测试
await runThumbnailTests();
```

## 测试要求

- 系统需要有图片文件用于测试
- 缩略图路径需要有写入权限
- 后端缩略图服务需要正常运行

## 故障排除

如果测试失败，请检查：

1. 后端服务是否正常运行
2. 缩略图路径是否有写入权限
3. 系统是否有图片文件可以用于测试
4. 控制台是否有详细的错误信息
